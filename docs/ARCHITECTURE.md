# Архитектура

Проект состоит из двух независимых частей.

## Клиент (`client/`)

Без сборщика — браузер подключает модули тегами `<script>` в порядке зависимостей.
Слои:

| Папка | Назначение |
|-------|------------|
| `js/core/config.js` | Единая конфигурация (`APP_CONFIG`) |
| `js/data/` | Справочные данные: категории, товары, изображения, теги |
| `js/core/` | Состояние (`S`), хранилище (`localStorage`), утилиты, рендер, навигация, API |
| `js/services/` | Бизнес-логика (питание/нормы) |
| `js/components/` | Переиспользуемые UI-элементы (карточки и пр.) |
| `js/ui/` | Нижние шторки |
| `js/screens/` | Экраны приложения |
| `js/features/` | Функциональные надстройки (магазин, лояльность, карты и т. д.) |
| `js/app.js` | Инициализация после загрузки всех модулей |

Порядок подключения важен: `config → data → core → services → components → screens → features → app → api`.
Инициализирующий вызов `render()` вынесен в `js/app.js`, чтобы выполниться после загрузки всех модулей.

## Сервер (`server/`)

Node.js + Express, классическая многослойная схема:

```
Запрос → middleware (cors, logger) → routes → controllers → services → JSON-хранилище
                                                models (валидация)
         errorHandler (обработка ошибок)
```

## Поток данных

Клиент работает на `localStorage`. Если сервер доступен — `js/core/api.js` загружает каталог из API
и отправляет оформленные заказы; иначе используется встроенный набор данных (безопасный фолбэк).
