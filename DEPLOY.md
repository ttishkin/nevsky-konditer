# Публикация приложения по ссылке (деплой)

Цель: получить публичный адрес вида `https://nevsky-konditer.onrender.com`, чтобы открыть
приложение с любого устройства. Работает всё: каталог, вход, заказы, дневник, админка, БД.

В проекте уже готово: `Dockerfile`, `render.yaml` (Blueprint), `.dockerignore`. Тебе остаётся
создать бесплатные аккаунты и нажать несколько кнопок (создавать аккаунты за тебя я не могу).

---

## Способ 1 — Render (рекомендую)

### Шаг 1. Выложить код на GitHub (бесплатно)
1. Зарегистрируйся на **github.com** (если ещё нет).
2. **New repository** → имя `nevsky-konditer` → **Create**.
3. На странице репозитория: **Add file → Upload files** → перетащи **содержимое папки
   `nevsky-konditer-2.0`** (папки `client`, `server`, файлы `Dockerfile`, `render.yaml`,
   `package.json`, `README.md` и т. д.). В папке нет `node_modules`, так что загрузка лёгкая.
4. **Commit changes**.

### Шаг 2. Развернуть на Render (бесплатно)
1. Зарегистрируйся на **render.com** (можно через тот же GitHub — кнопка «Sign in with GitHub»).
2. **New + → Blueprint** → выбери репозиторий `nevsky-konditer` → **Apply**.
   Render прочитает `render.yaml`, соберёт Docker-образ и запустит сервис.
   *(Альтернатива без Blueprint: New + → Web Service → выбрать репозиторий → Render сам найдёт
   `Dockerfile` → план **Free** → Create.)*
3. Дождись сборки (≈ 3–5 минут). Появится публичный адрес, например
   `https://nevsky-konditer.onrender.com`.

### Шаг 3. Открыть
- Приложение: `https://…onrender.com`
- Админ-панель: `https://…onrender.com/admin` (`admin@nk.ru` / `admin1234`)
- Документация API: `https://…onrender.com/api/docs`
- Демо-вход в приложении: `demo@nk.ru` / `demo1234`

---

## Важно знать про бесплатный тариф
- **«Засыпание»:** бесплатный сервис засыпает после 15 минут простоя — первое открытие после
  паузы грузится 30–60 секунд (это нормально, дальше быстро).
- **База данных:** на бесплатном тарифе диск временный — при перезапуске/передеплое база SQLite
  пересоздаётся. Каталог и демо-аккаунты (demo/admin) восстанавливаются автоматически, а
  созданные заказы могут обнулиться. Для постоянного хранения: подключить платный диск Render
  (смонтировать на `/app/server/data`) или перейти на управляемый PostgreSQL.
- **Карта:** для compute-сервисов Render может попросить привязать карту для верификации —
  списаний на бесплатном тарифе нет.

## Способ 2 — Koyeb (альтернатива, тоже бесплатно, Docker)
1. Зарегистрируйся на **koyeb.com**.
2. **Create Service → GitHub** → выбери репозиторий → Koyeb определит `Dockerfile` → **Deploy**.
3. Получишь адрес вида `https://…koyeb.app`.

---

## Если нужна только «витрина» интерфейса (без сервера)
Можно бесплатно и без аккаунта выложить **только клиент** (UI в демо-режиме на localStorage):
перетащи папку `client` на **app.netlify.com/drop**. Вход/админка/синхронизация по такой ссылке
работать не будут — только просмотр интерфейса. Ссылка временная.
