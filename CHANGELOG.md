# Изменения

Формат основан на «Keep a Changelog».

### [2.4.0] — 2026-05

**Добавлено**
- Swagger/OpenAPI: интерактивная документация API на `/api/docs`.
- Безопасность: helmet (security-заголовки) и express-rate-limit.
- Docker: `Dockerfile` + `docker-compose.yml` (запуск одной командой).
- Полноценный дневник: поиск по каталогу и добавление произвольного «своего продукта»; учёт в итогах и синхронизация.
- Лента уведомлений пользователя (`/api/notifications`) в профиле и напоминание заполнить дневник.

### [2.3.0] — 2026-05

**Добавлено**
- Реальная отправка писем через nodemailer (SMTP из .env; без настройки — лог; опц. Ethereal для демо).
- Админка: раздел «Пользователи» со списком и карточкой пользователя (профиль + его заказы).
- Админка: дашборд-график продаж по дням (SVG, без внешних библиотек).
- Эндпоинты: `/admin/users`, `/admin/users/:id`, `/admin/sales`.

### [2.2.0] — 2026-05

**Добавлено**
- Статус заказа из БД в истории клиента (включая «Отменён»).
- Админ-панель: пагинация, карточка деталей заказа, поиск, фильтр по статусу, выгрузка в CSV.
- Уведомления при смене статуса заказа (демо-почта: лог сервера + таблица notifications), эндпоинт `/admin/notifications`.

### [2.1.0] — 2026-05

**Добавлено**
- Авторизация в приложении: экран входа/регистрации, сессия (JWT), вход в профиле.
- Двусторонняя синхронизация с сервером: при входе подгружаются дневник, заказы, корзина и избранное; изменения зеркалятся на сервер.
- Статусы заказов (new/processing/shipped/delivered/cancelled).
- Админ-панель фабрики (`/admin`): просмотр всех заказов из БД, смена статуса, сводная статистика.
- Эндпоинты: `/cart`, `/favorites`, `/admin/orders`, `/admin/orders/:id/status`, `/admin/stats`.

## [1.0.0] — 2026-05

### Добавлено
- Онбординг с расчётом суточной нормы (Миффлин–Сан Жеор) и «бюджетом на сладкое».
- Каталог: сторис, баннеры, товар дня, полки по цели, наборы, поиск с подсказками, фильтры и сортировка.
- Карточка товара: галерея, КБЖУ, состав, отзывы, похожие товары.
- Дневник питания: кольцо прогресса, добавление в штуках, редактирование количества.
- Корзина и оформление: доставка, промокод, списание бонусов, Яндекс-карты.
- Программа лояльности и магазин за баллы.
- Светлая и тёмная темы.
- Бэкенд (Node.js + Express): API каталога и заказов, файловое JSON-хранилище.

### Архитектура
- Клиент разнесён по модулям: data / core / services / components / ui / screens / features.
- Сервер по слоям: routes → controllers → services → models, middleware и utils.
